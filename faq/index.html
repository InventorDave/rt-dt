  • How do I load resources to make the later sample scenes work?

    Later options in the "sample scenes" drop-down box require some support files to be loaded.

    To load jpg/png/bmp textures (called Maps in RT-DT), select File->Load Image File.
    (The console will output a message when the image file has been loaded.)

    To load .PPM files (ASCII format only!), select File->Load PPM File. Again, the console will tell you when it has loaded the file.

    In the File Menu is an option to download PPM files from our server, whence you can then load them into the raytracer. You should especially note bgImage/bgImage2.ppm, 2k_moon.ppm and the 3 earth PPM's, for the 'Earth & Moon' sample scene. See below for how to configure the raytracer to use the files as 'Maps' and a 'BG Image'.


    The options you have are to:

    1. Create named 'Maps', by selecting Options->Set Maps, and using the dialog to name a Map (hint: name one 'earth', no quotes), then from the drop-down boxes, use the upper one to select a Map name you've created, then use the lower drop-down box to select a texture you've already loaded (image file).

    2. Create a 'Normal Map', for producing perturbed surfaces (such as an Earth with raised mountains - sample scene 'Bump Scene'), by loading a suitable NORMAL map in a suitable image format (not actually a 'bump' map - they encode the data differently), then going to Options->Set/Unset Normal Map, and simply selecting the loaded image file you want to use.

    3. Set the 6 sides of a 'Skybox' (where the camera is inside the cube, so the cube sides provide a panoramic wrap-around environment, which you can look at by rotating the virtual Camera. Set up the Skybox by first loading up to 6 (1 is fine!) image files, then going to Options->Set Skybox, and setting an image for each of the 6 sides via the 6 drop-down boxes. Then the sample scene 'Sky Box Test' will work.
